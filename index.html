Quiz 1 — Sample Question
A development team is designing a new e-commerce application. They want to ensure that when an order is placed, multiple downstream services (inventory, shipping, payment) are notified independently without the order service knowing about each consumer. Which architectural pattern best fits this requirement?
  • ✓ Choreography with event-driven architecture
  • Microservices with synchronous REST APIs
  • Monolithic architecture with shared database
  • Orchestration with a central coordinator
Show Explanation
✓ Why correctChoreography with event-driven architecture allows the order service to publish an event without knowing about consumers. Each service subscribes to events independently, achieving loose coupling.
✗ Why A is wrongThis is the correct answer.
✗ Why B is wrongSynchronous REST APIs create tight coupling because the order service must know about and directly call each downstream service, waiting for each response before proceeding.
✗ Why C is wrongMonolithic architecture doesn't provide the independent service notification pattern required and bundles all logic into a single deployable unit.
✗ Why D is wrongOrchestration uses a central coordinator that must know about all downstream services and manage the flow, creating tighter coupling than choreography where services react independently to events.
Quiz 2 — Sample Question
A developer is writing a Lambda function that must query an Amazon RDS PostgreSQL database in a private subnet. The function currently times out when attempting to connect. What should the developer do to allow the Lambda function to access the database?
  • Make the RDS instance publicly accessible and connect over the internet
  • Attach an internet gateway to the VPC and route Lambda traffic through it
  • ✓ Configure the Lambda function with the VPC private subnets and a security group that allows access to the RDS instance
  • Increase the Lambda function timeout to 15 minutes
Show Explanation
✓ Why correctTo access resources in a private VPC subnet, a Lambda function must be configured with VPC settings — specifically, it needs at least one private subnet and a security group. AWS creates Elastic Network Interfaces (ENIs) in the specified subnets, allowing the function to communicate with VPC resources like RDS. The Lambda execution role also needs the ec2:CreateNetworkInterface, ec2:DescribeNetworkInterfaces, and ec2:DeleteNetworkInterface permissions (included in the AWSLambdaVPCAccessExecutionRole managed policy).
✗ Why A is wrongMaking the RDS instance publicly accessible defeats the purpose of placing it in a private subnet and introduces a security risk. Lambda should connect via VPC configuration, not over the public internet.
✗ Why B is wrongAn internet gateway allows traffic to/from the internet for public subnets. It does not help Lambda reach a private subnet resource. Lambda needs VPC attachment, not internet routing.
✗ Why D is wrongIncreasing the Lambda timeout may delay the error but does not resolve the root cause — the function has no network path to the RDS instance without VPC configuration.
Quiz 3 — Sample Question
A developer is designing a DynamoDB table for a multi-tenant SaaS application that stores user activity logs. The table will handle millions of writes per day across thousands of tenants. Which TWO partition key strategies provide the BEST balanced partition access?
  • Use the current date as the partition key
  • ✓ Use a composite key of tenantId#activityTimestamp as the partition key
  • Use a boolean "isActive" field as the partition key
  • Use the AWS region name as the partition key
  • ✓ Use a randomly generated UUID as the partition key
Show Explanation
✗ Why B and E are correctA composite key of tenantId#activityTimestamp provides high cardinality because the combination of tenant ID and timestamp creates many unique values, distributing data evenly across partitions. A UUID partition key provides the highest possible cardinality, ensuring perfectly even distribution. Both strategies avoid hot partitions by maximizing the number of distinct partition key values.
✗ Why A is wrongUsing the current date creates a hot partition because all writes for a given day go to the same partition. With millions of daily writes, this causes throttling.
✗ Why C is wrongA boolean field has only two possible values (true/false), creating extremely low cardinality and severe hot partitions.
✗ Why D is wrongAWS region names provide very few distinct values (roughly 30), resulting in low cardinality and uneven data distribution.

  • ✓ Use Lambda layers for shared dependencies
  • Download dependencies at runtime on each invocation from the package registry
Show Explanation
✓ Why correctLambda layers allow you to package dependencies separately (up to 250 MB unzipped total across layers) and share them across multiple functions. This reduces each function's deployment package size and enables reuse without duplicating dependencies.
✗ Why A is wrongIncluding all 100 MB of dependencies in every deployment package works functionally but inflates each package, slows deployments, and prevents reuse across functions. Layers solve all three issues.
✗ Why B is wrongDownloading dependencies at runtime on each invocation adds significant latency to every cold and warm start, increases execution time costs, and may fail due to network issues or rate limiting from package registries.

Quiz 6 — Sample Question
A developer stores sensitive customer data in Amazon S3. A security audit requires that all data transmitted between the application and S3 must be encrypted. Which action ensures encryption in transit for all S3 requests?
  • ✓ Enforce aws:SecureTransport condition in the S3 bucket policy
  • Enable server-side encryption with SSE-S3
  • Enable S3 Block Public Access
  • Enable S3 Versioning on the bucket
Show Explanation
✓ Why correctEncryption in transit protects data as it moves between the application and the S3 endpoint. Enforcing aws:SecureTransport in a bucket policy denies any request that does not use HTTPS (TLS), ensuring all data is encrypted while traveling over the network.
✗ Why B is wrongServer-side encryption (SSE-S3, SSE-KMS) protects data at rest on disk, not while it is being transmitted over the network. Even with SSE enabled, data could still be sent over an unencrypted HTTP connection without a SecureTransport policy.
✗ Why C is wrongS3 Block Public Access prevents public anonymous access to buckets and objects. It does not enforce TLS for authenticated API calls, so data could still travel unencrypted between authorized clients and S3.
✗ Why D is wrongS3 Versioning preserves previous versions of objects for recovery and audit purposes. It has no effect on how data is transmitted and does not enforce encryption in transit.
Quiz 7 — Sample Question

✓ Why correctAWS Secrets Manager is purpose-built for managing sensitive credentials. It supports automatic rotation of database credentials using built-in Lambda rotation functions, encrypts secrets at rest using AWS KMS, and provides fine-grained IAM access control. The Lambda function retrieves the current credentials at runtime, so rotated credentials are picked up automatically without redeployment.

Domain 4: Troubleshooting and Optimization
A development team runs a web application on an Auto Scaling group behind an Application Load Balancer. Traffic increases gradually during business hours and drops at night. The team wants to maintain response latency below 100 ms while minimizing costs. Which Auto Scaling policy type should the developer configure?
  • Simple scaling policy triggered by a CloudWatch alarm on CPU > 70%
  • ✓ Target tracking scaling policy that maintains average CPU utilization at 50%
  • Scheduled scaling that adds instances at 8 AM and removes them at 6 PM
  • Step scaling policy with multiple thresholds at 40%, 60%, and 80% CPU
Show Explanation
✓ Why correctTarget tracking scaling is the simplest and most effective approach for gradual, predictable traffic patterns. It automatically adjusts capacity to maintain the target metric (e.g., 50% CPU), scaling out as traffic rises and scaling in as it drops — without needing to define individual alarms or thresholds. This matches the gradual traffic pattern and keeps latency low.
✗ Why A is wrongSimple scaling reacts to a single alarm threshold and has a cooldown period during which no further scaling occurs. This can cause latency spikes because scaling lags behind gradual load increases.
✗ Why C is wrongScheduled scaling uses fixed times, which doesn't account for actual demand variations. If traffic patterns shift (e.g., a holiday), the schedule won't adapt, leading to either over-provisioning (higher cost) or under-provisioning (latency spikes).
✗ Why D is wrongStep scaling is useful for unpredictable, spiky traffic patterns where you need different scaling responses at different thresholds. For gradual, predictable increases, target tracking is simpler and more cost-efficient because AWS manages the scaling math automatically.
Quiz 20 — Sample Question
A developer is designing a DynamoDB table to store customer orders. The application must efficiently retrieve all orders for a specific customer sorted by date, and the table will handle thousands of writes per second across many customers. Which TWO design decisions should the developer make?
  • Use a sequential numeric ID as the partition key to keep items ordered
  • ✓ Use a composite primary key with CustomerId as partition key and OrderDate as sort key
  • Scan the table with a filter expression on CustomerId
  • ✓ Use a high-cardinality attribute as the partition key to ensure even data distribution
  • Use OrderStatus as the sole partition key for simple lookups
Show Explanation
✓ Why correctA composite primary key (partition key + sort key) allows multiple items to share the same partition key while being differentiated by the sort key. This enables efficient queries that retrieve all related items (e.g., all orders for a customer) using a single Query operation on the partition key. A high-cardinality partition key (such as CustomerId) distributes data evenly across partitions, preventing hot partitions and throttling.
✗ Why A is wrongUsing a sequential numeric ID (e.g., auto-incrementing) as the partition key creates hot partitions because DynamoDB assigns partitions based on key hash values. Sequential IDs cause writes to concentrate on a single partition, leading to throttling under high throughput.
✓ Why B is wrongA composite key with CustomerId as partition key and OrderDate as sort key is correct — it allows querying all orders for a customer sorted by date using a single Query operation.
✗ Why C is wrongA single attribute primary key (partition key only) with a low-cardinality attribute like OrderStatus would create hot partitions because most items would cluster under a few values (e.g., "Pending", "Shipped"). This is a poor key design.
✗ Why D is wrongHigh-cardinality partition keys are correct — they ensure even data distribution across DynamoDB partitions, which is a fundamental best practice for key design.
✗ Why E is wrongScan with a filter expression reads the entire table and then discards non-matching items. This is expensive in terms of read capacity and latency compared to a Query on a well-designed composite key.
Quiz 21 — Sample Question
A developer's application uses an Amazon SQS standard queue. Consumers occasionally process the same message twice because it becomes visible again before the first consumer finishes. What should the developer do to prevent this?
  • Increase the SQS message retention period to 14 days
  • ✓ Increase the visibility timeout to longer than the average processing time
  • Enable SQS long polling with a 20-second wait time
  • Switch the queue from standard to FIFO
Show Explanation
✓ Why correct (B)When the visibility timeout is shorter than the time it takes a consumer to process a message, the message becomes visible again and is delivered to another consumer before the first one finishes. Increasing the visibility timeout beyond the average processing time prevents duplicate processing.
✗ Why A is wrongMessage retention period controls how long unprocessed messages stay in the queue (default 4 days, max 14 days). It does not affect whether messages are redelivered to other consumers during processing.
✗ Why C is wrongLong polling reduces empty responses and API costs by waiting up to 20 seconds for messages to arrive before returning. It does not affect message visibility or duplicate delivery.
✗ Why D is wrongFIFO queues guarantee ordering and exactly-once processing, but switching queue types does not address the root cause—if the visibility timeout is too short, messages will still be retried. Additionally, FIFO queues have a throughput limit of 300 messages/second (3,000 with batching) which may not suit all workloads.

  • Use CloudFormation Mappings with a FindInMap function keyed by environment name
  • ✓ Use CloudFormation dynamic references to resolve values from AWS Systems Manager Parameter Store at deploy time
  • Export values from one stack and import them in another using CloudFormation cross-stack references
Show Explanation
✓ Why correctCloudFormation dynamic references (using the {{resolve:ssm:parameter-name:version}} syntax) allow templates to retrieve values from Systems Manager Parameter Store at stack creation or update time. The developer stores environment-specific values (database strings, API URLs) in Parameter Store with a naming convention like /app/prod/db-connection and /app/dev/db-connection, then references them dynamically in the template. This keeps the template environment-agnostic, centralizes configuration in Parameter Store, and supports SecureString for sensitive values using {{resolve:ssm-secure:...}}.

✗ Why B is wrongCloudFormation Mappings with FindInMap works for a small, fixed set of values known at template authoring time (like AMI IDs per region). Cross-stack references also create tight coupling — an exported value cannot be modified while any stack imports it.
